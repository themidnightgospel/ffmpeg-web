The system behind every tool

Design System.

One Swiss-typographic language — white paper, near-black ink, a single signal-red accent, hairline rules, sharp corners. Every token and component below lives in ffmpeg-web.css and is shared across the hub and all 54 tool pages.

Foundations

Principles

Restraint over decoration. Flat surfaces, no shadows, no gradients, no rounded corners. Structure comes from hairline rules and a strict grid.

One accent, used deliberately. Signal red marks only what matters: links, the active state, focus, and progress. Never as decoration.

Type does the work. A single grotesque (Inter) at a tight display scale, with uppercase tracked-out labels for hierarchy.

Accessible by default. Real semantics, visible focus rings, keyboard support, and reduced-motion respect are built into the components.

Layout

Content is capped at --measure 1180px and inset by a fluid --gutter. Tool pages use a two-column .work grid (source · settings) that collapses on mobile. Structure is drawn with hairlines, never boxes with shadows.

Components

Segmented control

The house style for single-choice options — preferred over native dropdowns. Accessible radio inputs styled as outlined buttons; selected fills ink, hover/focus is red. Markup: label.seg > input[type=radio] + span inside .segmented[role=radiogroup].

Inputs & fields

Text / number / textarea inputs for values a button set can't express — timestamps, dimensions, filenames. Each sits in a .field with an uppercase .flabel.

Components

Time field

Structured HH : MM : SS . mmm entry for trim in/out points, timestamps, and seeks — used by the Trimmer, Splitter, Thumbnail, and any tool that needs a precise moment. Each segment accepts digits only, clamps to range, zero-pads on blur, and auto-advances; arrow keys and Tab move between segments. Wrap in a .field with a .flabel like any other control.
